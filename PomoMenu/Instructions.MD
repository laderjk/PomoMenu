
# Pomodoro Menu Bar App вҖ” Build Spec for Claude Code

## Goal
Build a native macOS Pomodoro timer app in SwiftUI with a `MenuBarExtra` front-end, Slack status + DND sync, and local CSV stats. Target macOS 26+ (Tahoe) so `MenuBarExtra` with `.window` style is available. No App Store distribution вҖ” ship as a signed (ad-hoc is fine) `.dmg` and optionally a Homebrew cask.

I have Xcode installed. Scaffold the whole project, build it, and leave me with a runnable `.app` and a `Makefile`/script that produces a `.dmg`.

## Tech constraints
- **Language/UI:** Swift 5.9+, SwiftUI only. No AppKit except where unavoidable (e.g., `NSWorkspace` for opening URLs, `LSUIElement` for hiding the Dock icon).
- **Architecture:** MVVM. One `TimerEngine` (ObservableObject) as the single source of truth, injected into the menu bar view and the settings window.
- **Persistence:** 
  - Settings вҶ’ `UserDefaults` via `@AppStorage`.
  - Stats вҶ’ CSV at `~/Library/Application Support/Pomo/stats.csv` (create dir if missing).
  - Slack token вҶ’ **Keychain** (never UserDefaults, never CSV, never logs).
- **Concurrency:** Timer driven by a single `Task` loop using `ContinuousClock` / `Date` deltas вҖ” **not** `Timer.scheduledTimer`, so it stays accurate if the app is backgrounded or the Mac sleeps. On resume, recompute remaining time from the phase's absolute end date.
- **App shell:** `LSUIElement = true` in Info.plist (menu bar only, no Dock icon). Settings window opens via `Settings { ... }` scene.

## Session model

```swift
enum SessionType {
    case regularFocus     // 25 min, рҹҚ…
    case deepFocus        // 50 min, рҹ§ 
    case shortBreak       // 5 min,  вҳ•пёҸ
    case longBreak        // configurable, рҹҢҝ
}
```

- **Default cycle:** 4Г— (regular focus вҶ’ short break) вҶ’ long break вҶ’ repeat.
- **Deep focus** is a one-off the user triggers manually; it's followed by a 15-min break (configurable) and does **not** count toward the 4-pomo cycle by default (expose a toggle: "Count deep focus toward cycle").
- **Skip:** every phase has a Skip action. Skipping ends the phase immediately, logs it as `skipped`, and advances to the next phase.
- **Pause/Resume** and **Reset cycle** also required.

## Settings (all live-editable, persisted via `@AppStorage`)
- Regular focus duration (default 25 min)

  - Current session type + time remaining (large).

## Slack integration (user token approach)

Use a Slack **user token** (xoxp-вҖҰ) created via a personal Slack app with these user scopes: `users.profile:write`, `dnd:write`. Store in Keychain. Provide a settings field + a "Test connection" button that calls `auth.test`.

Endpoints to call (all `POST`, `Authorization: Bearer <token>`, JSON body):

| Event | Call |
|---|---|
| Regular focus start | `users.profile.set` вҶ’ status_text="Focusing", status_emoji=":tomato:", status_expiration=end_epoch |
| Deep focus start | `users.profile.set` (рҹ§  / "Deep focus вҖ” do not disturb") **and** `dnd.setSnooze?num_minutes=<duration>` |
| Short break start | `users.profile.set` (вҳ•пёҸ / "On a break") |
| Long break start | `users.profile.set` (рҹҢҝ / "On a long break") |
| Any session end / app quit | `users.profile.set` with empty status; if DND was set, `dnd.endSnooze` |

Make Slack calls **fire-and-forget** on a background task; never block the timer. Log failures locally (don't surface scary errors in the menu bar вҖ” a small вҡ пёҺ badge in the popover is enough). Emojis are user-editable in settings; the defaults above are sensible starting points.

Put all Slack code behind a `SlackClient` protocol so it's trivial to stub in tests.

## Stats CSV

Path: `~/Library/Application Support/Pomo/stats.csv`. Append-only. Header row on first write:

```
date,start_time,end_time,type,planned_minutes,actual_minutes,status,task
```

- `date` вҶ’ `YYYY-MM-DD` in local tz
- `start_time`, `end_time` вҶ’ ISO-8601 local
- `type` вҶ’ `regular|deep|short_break|long_break`
- `status` вҶ’ `completed|skipped`
- `task` вҶ’ optional free text; prompt for it (optional, dismissable) when starting a focus session вҖ” **not** on breaks. Escape commas/quotes properly (wrap in quotes, double internal quotes).

Expose a "Reveal stats in Finder" button in settings.

A couple of notes on why I drafted it this way, in case you want to tweak before pasting:

- I pinned the **timer to wall-clock deltas** rather than tick counting вҖ” this is the single most common way these apps go wrong when the Mac sleeps mid-pomo.
- I made the Slack token **Keychain-only** and ring-fenced behind a protocol; it's a nice boundary and makes the tests clean.
- I left the **long-break default at 15 min** but fully configurable, per your answer.
